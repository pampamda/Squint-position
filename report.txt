X.2 Multi-Shape Primitive Learning

X.2.1 Simulation Environment and Task Setup

All training is conducted in ManiSkill3 on top of the SAPIEN physics engine, which supports fully GPU-parallelized simulation. Each training run uses 1024 parallel environments, enabling high-throughput experience collection at approximately 638 simulation steps per second on a single consumer GPU. The robot is an SO-101 six-degree-of-freedom desktop manipulator controlled through a pd_joint_target_delta_pos interface at 15 Hz during training, with a wrist-mounted RGB camera rendering at 128×128 pixels as the primary visual input.

The task suite is structured around three primitives — pick, place, and stack — each trained as an independent policy that learns a closed-loop visuomotor mapping from the current observation to a joint-space delta command. To promote shape-invariant grasping and manipulation, four distinct object geometries are included in every training environment: cube, sphere, capsule, and cylinder. These geometries span a representative range of contact affordances and inertial properties, providing sufficient coverage to validate the multi-shape generalization hypothesis without exhaustive enumeration.

X.2.2 Observation Space, Policy Architecture, and Shape Conditioning

Each environment produces a mixed observation consisting of a raw RGB image from the wrist camera and a structured state vector. The state vector encodes the robot's proprioceptive information (joint positions, joint velocities, TCP pose, distance to the rest configuration), task-specific features that vary per primitive (such as estimated object position for pick, or target coordinates for place), and a four-dimensional one-hot vector identifying the current object geometry. This shape identity field, item_shape_id, is the mechanism by which a single policy conditions its behavior on the object class without separate network branches.

The visual stream is processed by a compact convolutional encoder that maps the 16×16 downsampled image through a series of convolutional layers to a 1024-dimensional feature vector, then projected to a 50-dimensional visual embedding with LayerNorm and Tanh activation. The state vector is separately projected by a linear layer to 256 dimensions followed by LayerNorm and ReLU. These two representations are concatenated to form a 306-dimensional joint feature that is shared between the actor and critic.

Training uses Soft Actor-Critic (SAC) with a C51 distributional critic. The critic maintains 101 return atoms spanning the range [-20, 20], allowing it to represent multi-modal return distributions that arise from the staged reward shaping used across all three primitives. To reduce critic variance, the Q-ensemble is implemented with vmap parallelism rather than separate network copies. The actor is a three-layer fully connected network with 256 units per layer and LayerNorm, outputting the parameters of a squashed Gaussian over joint-space actions.

The design choice to encode shape identity as a one-hot in the state rather than as a separate network input or a modulation signal is intentional. Because the state projection concatenates this identity information early into the joint feature, the actor and critic see shape context at every internal computation. This allows the shared network to develop geometry-specific internal representations without the overhead of conditional normalization or adapter modules, which would complicate the sim-to-real transfer pipeline.

X.2.3 Parallel Environment Shape Assignment and Training Strategy

In each parallel training environment, the object geometry is assigned deterministically by computing env_idx modulo 4 and mapping the result to one of the four shape classes. This round-robin assignment ensures that every batch of gradient updates contains experience from all four geometries in equal proportion. The assignment is fixed per environment index and does not change across episodes, so the replay buffer accumulates balanced coverage automatically without any explicit sampling correction.

For the pick primitive, training begins from a warm-started checkpoint originally trained on cube-only grasping. The weight transfer is performed by patch_ckpt.py, which copies the convolutional encoder and all fully connected layers from the single-shape checkpoint while re-initializing the state projection's first weight matrix to accommodate the expanded state dimension (which now includes item_shape_id). This warm-start accelerates convergence on the shared visual and motor representations while allowing the policy to adapt its state processing to the new geometry-conditioning signal.

The place and stack primitives are trained from scratch on SO101PlacePosMultiShape-v1, a new environment introduced in this work. Unlike the earlier bin-based place environment, PlacePosMultiShape provides a continuous target position sampled uniformly on the table surface during each episode reset. The policy receives the target as part of the state observation and must learn to carry the held object to an arbitrary coordinate rather than classifying into discrete bins. This formulation makes the same checkpoint reusable for stack: the PrimitiveExecutor passes a target position elevated above the surface of a destination object, and the place policy generalizes to this elevated goal without any additional training.

X.2.4 Preliminary Results and Current Limitations

Training of the multi-shape pick policy (SO101LiftMultiShape-v1) converged to qualitatively reliable grasping behavior across all four object geometries. The policy correctly adapts its approach trajectory and gripper timing depending on the shape identity provided in the observation, demonstrating that the one-hot conditioning is effective at routing geometry-specific motor programs through the shared network.

As an intermediate validation of the coordinate-based place formulation, the earlier bin-based policy (SO101PlaceMultiShape-v1) achieved a success rate of 69% (success_once = 0.69) after 1.5 million training steps in approximately 39 minutes of wall-clock time. This result establishes that the multi-shape place task is learnable within the SQUINT training framework and provides a performance baseline for the coordinate-based variant.

The primary current limitation of the multi-shape training pipeline is that shape diversity is achieved through fixed geometry parameterizations rather than continuous randomization over geometry parameters such as scale or aspect ratio. The four chosen shapes cover qualitatively distinct contact properties, but a policy trained on this discrete set may still fail on objects whose geometry falls between these categories. A second limitation is that the shape identity is provided as ground-truth one-hot in both training and deployment; in an end-to-end real-world system, this identity would need to be inferred from the visual observation or provided by an external recognition module.


X.3 Primitive Composition and Unified Execution Interface

The three learned primitives — pick, place, and stack — are integrated into a single PrimitiveExecutor class that exposes a planner-facing API and handles all sim-to-real bridging internally. Each primitive runs its own ManiSkill3 simulation instance that maintains the robot's kinematic state across real-world steps. The executor is designed so that state is compatible across consecutive primitive invocations: when pick completes with the object held in the gripper, the transition to place does not reset the real robot or interrupt the grasp. Instead, only the simulation environment for the place policy is (re-)initialized, and the gripper state observed by the place policy at the first step reflects the actual physical grasp established by pick. This state-compatible handoff is achieved through a custom _place_hold_reset function that re-initializes the sim without issuing any motion commands to the real hardware, preserving the grasp between primitives.

From the planner's perspective, each primitive presents a clean, geometry-agnostic interface: pick(object_pos) accepts a world-frame position estimate and executes a grasp, place(target_pos) accepts a world-frame goal coordinate and releases the held object there, and stack(target_pos) is an alias for place with the caller responsible for computing the appropriate height above the destination surface. The planner is not required to know which object shape is present or which network checkpoint is active; shape identity is read from the simulation state and injected into the policy's observation automatically. This abstraction means that a task planner operating over symbolic predicates — Grasped(obj), At(obj, loc), Stacked(obj1, obj2) — can be grounded to motor behavior by mapping each predicate-achieving action to a single executor call, without any geometry-specific dispatch logic in the planning layer.

The two axes of generalization — object geometry and target coordinate — form a capability map that covers the core manipulation operations needed for tabletop rearrangement tasks. The pick policy generalizes across four geometry classes through shape conditioning; the place and stack policies generalize over continuous goal positions through coordinate injection at deploy time. Together, these two axes allow the system to handle pick-and-place and stacking tasks for arbitrary combinations of object shape and placement goal within the training distribution, using only three learned checkpoints and no task-specific fine-tuning. The stack primitive in particular demonstrates the leverage of this design: by reusing the place policy rather than training a separate stacking policy, the system achieves a three-primitive task repertoire at the cost of two trained networks, and the place policy's generalization to elevated goals emerges from the continuous coordinate conditioning rather than from explicit stack-specific reward shaping.
